# 《404号房间》户外场景交互映射说明（开发者版）

## 1. 文档用途

本文档定义三张户外背景与可交互热区、浮层素材（`images/scenes/`）、业务规则的对应关系。结构与室内版 `场景交互映射.md` 一致，供前端挂载热区、切换场景、弹出浮层时直接对照。

**赛博世界观**：户外场景不追求现实地理逻辑。树林可以是「长在地上的缓存区」，城市是「霓虹协议街区」，外太空是「404 房间漂出窗外的数据虚空」。`scenes/` 下的像素小图标仅在 L4 浮层内临时展示，主界面不常驻。

---

## 0. 核心展示原则（与室内一致）

| 允许常驻展示 | 不允许常驻展示 |
| --- | --- |
| 背景环境彩绘（含树桩/贩卖机/火箭等平台轮廓） | `scenes/` 下全部独立元素贴图 |
| 桌宠小人、隐藏小猫 | 浮层道具、解锁装饰、藏品图标散落 |
| 全局 UI（余额、收藏柜、设置、场景 Tab） | 街机 PLAY 闪烁标、打开态冰箱/桶盖 |

**交互元素一律在点击热区后于浮层内临时展示；关闭后主界面恢复干净。**

## 4. 场景切换规则

| 项目 | 规则 |
| --- | --- |
| Tab 组 | 底部展示「数据林海」「霓虹街区」「数据虚空」三 Tab；当前高亮 |
| 与室内关系 | 若产品保留室内双房间，建议 UI 分两级：「室内 / 户外」大 Tab，户外下再切三子 Tab；本文仅定义户外三图 |

| 状态保留 | 切换不重置桌宠状态、不中断 Buff、不重复离线结算 |

| --- | --- |
