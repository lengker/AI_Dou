# 《404号房间》场景交互映射说明（开发者版）

## 1. 文档用途

本文档定义两张房间背景与全部可交互元素、浮层素材、业务规则的对应关系，供前端开发挂载热区、切换场景、弹出浮层时直接对照。

业务规则以 `PRD.MD` 为准；本文档只说明「点哪里、出现什么、调用哪条 PRD 规则」。

---

## 0. 核心展示原则（必读）

**主界面常态不挂载任何可交互元素贴图。** 房间主界面默认只展示以下内容：

| 允许常驻展示 | 不允许常驻展示 |
| --- | --- |
| 背景图（环境彩绘，含床/桌/冰箱等环境轮廓） | `interact/`、`element/`、`collectible/` 下的独立元素贴图 |
| 桌宠小人 | 电脑浮层素材、可乐、咖啡杯、地毯、夜灯、霓虹贴纸、书架摆件 |
| 全局 UI（余额、收藏柜、设置、房间 Tab） | 街机 PLAY 闪烁标、垃圾桶打开态、冰箱内部 |
| 状态反馈（思念气泡、表情替换，有时效） | 已解锁家具的持久贴图叠加 |
| 隐藏小猫（T1 解锁后，跟随小人） | 藏品图标在房间内散落展示 |

**交互元素一律在点击热区后，于浮层 / 弹窗 / 全屏面板内临时展示；关闭浮层后元素消失，主界面恢复干净。**

背景图里画好的床、电脑轮廓等属于**环境彩绘**，不是可交互贴图，不承担元素展示职责。可点击区域仅为透明热区，用户看不到额外素材，直到点击触发浮层。

## 2. 背景资源

| 场景 ID | 背景资源名 | 场景名称 | 主题 |
| --- | --- | --- | --- |
| `room_working` | `background_working` | 办公区 | 双工位、电脑、书架、墙面办公装饰 |
| `room_living` | `background_living` | 生活区 | 床、冰箱、街机、垃圾桶、中央空地 |

两张背景共用同一套建筑壳：**左墙白色、右墙浅蓝色、橙色横纹木地板、右墙固定位置窗户**。像素风格、等距透视一致。

## 3. 双房间分工总览

| 交互类型 | 办公区 | 生活区 |
| --- | --- | --- |
| 电脑 | ✅ | — |
| 床 | — | ✅ |
| 冰箱 | — | ✅ |
| 垃圾桶 | — | ✅ |
| 街机（赛博跳跃入口） | — | ✅ |
| F01 桌子咖啡杯 | ✅ | — |
| F02 床头小夜灯 | — | ✅ |
| F03 地毯像素图案 | — | ✅ |
| F04 墙面霓虹贴纸 | ✅ | — |
| F05 书架杂物摆件 | ✅ | — |
| 收藏柜 / 设置 / 数据碎片余额 | 两房间均有（UI 层） | 同左 |
| 桌宠小人 / 隐藏小猫 | 两房间均有（角色层） | 同左 |

**原则**：PRD 定义的 4 个固定交互点（电脑、床、冰箱、垃圾桶）与 5 件可解锁家具，合计 9 项场景交互，由两张背景完整覆盖。所有交互元素**仅在点击后于浮层内展示**，主界面不预置贴图。

**背景与环境的分工**：背景图只提供空间定位（用户点哪里像是要互动）；`interact/`、`element/` 素材只在浮层中出现。

## 4. 房间切换规则

| 项目 | 规则 |
| --- | --- |
| 切换入口 | 房间主界面底部固定展示「办公区」「生活区」两个 Tab；当前房间 Tab 高亮 |
| 默认房间 | 用户首次进入房间主界面时，默认展示办公区 |
| 切换动画 | 0.3 秒横向滑动切换背景；桌宠同步位移至目标房间默认站立点 |
| 状态保留 | 切换房间不重置桌宠状态、不中断进行中的临时 Buff、不重复触发离线结算 |
| 办公区默认站立点 | 两工位之间的地板中央 |
| 生活区默认站立点 | 床与街机之间的地板中央（地毯热区上方） |

## 5. 图层结构（开发挂载顺序）

自底向上共 4 层常驻层 + 1 层按需层，禁止在主界面常驻层叠放交互元素贴图：

| 层级 | 名称 | 常驻 / 按需 | 内容 |
| --- | --- | --- | --- |
| L0 | 背景层 | 常驻 | `background_working` 或 `background_living` |
| L1 | 角色层 | 常驻 | 默认像素小人、隐藏小猫（T1 解锁后） |
| L2 | 热区层 | 常驻（透明） | 透明可点击区域；调试模式可显示边框，发布版不可见 |
| L3 | UI 层 | 常驻 | 数据碎片余额、Tab、收藏柜、设置入口 |
| L4 | 浮层层 | **按需** | 交互弹窗、解锁面板、气泡、表情/思念反馈；关闭即销毁，不留残影 |

**不存在「家具常驻层」**。F01—F05 解锁状态仅存数据；用户再次点击对应热区时，在 L4 浮层内展示该家具贴图。

## 11. 浮层弹窗统一规范

| 类型 | 关闭方式 | 动效 |
| --- | --- | --- |
| 交互浮层（电脑、冰箱、翻垃圾桶等） | 点击浮层外遮罩或右上角「关闭」 | 老式显像管横向压缩闪烁（PRD 6.1） |
| 解锁确认面板 | 「确认解锁」或「取消」 | 同上 |
| 装饰气泡 | 5 秒自动消失，或点击屏幕任意处关闭 | 打字机逐字（PRD 6.3） |
| 离线结算弹窗 | 点击「知道了」 | 打字机逐字 |

**原则**：所有 `interact/`、`element/`、家具类贴图**只在 L4 浮层存活期内展示**；浮层关闭 = 元素从屏幕移除。主界面始终保持：背景 + 小人 + 全局 UI。
